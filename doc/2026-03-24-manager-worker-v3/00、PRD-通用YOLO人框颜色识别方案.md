# PRD：通用 YOLO 人框颜色识别方案

## 1. 文档目的

本文档用于定义一套基于通用 YOLO 的轻量化视频识别方案。方案核心思路是：

- 先使用通用 YOLO 模型识别出画面中的人
- 再基于每个人的人体框，按大致人体结构位置裁出“安全帽颜色识别区域”和“反光衣颜色识别区域”
- 最后在这些区域内做颜色分析，输出帽子颜色、反光衣颜色及对应业务标签

该方案不依赖独立的安全帽检测模型或反光衣检测模型，优先追求落地快、工程简单、适合固定机位场景。

## 2. 业务背景

当前场景为固定机位夜间工地视频，目标是对施工人员进行基础安全着装识别。现阶段主要希望识别：

- 人员是否出现在画面中
- 安全帽的颜色
- 反光衣的颜色

当前问题包括：

- 夜间画面亮度不稳定
- 反光区域容易受强光和地面反射干扰
- 安全帽较小，且部分人员存在遮挡
- 若直接在整个人体框内做颜色识别，背景、裤子、皮肤和周边物体容易混入

因此，需要一个以“人框为基础、再做区域化颜色识别”的方案。

## 3. 产品目标

### 3.1 核心目标

- 使用通用 YOLO 稳定识别画面中的人
- 基于人框估计帽子区域和反光衣区域
- 在区域内完成颜色识别
- 输出结构化结果，支持视频标注与后续统计

### 3.2 目标特点

- 优先适配固定机位
- 优先适配当前夜间工地视频
- 优先使用规则方案，不引入额外训练成本
- 支持后续向“专用安全帽/反光衣检测模型”升级

### 3.3 非目标

当前阶段不作为核心目标的能力包括：

- 不做人脸识别
- 不做身份实名制识别
- 不做复杂遮挡场景下的高精度服饰分割
- 不做跨摄像头人员重识别
- 不以单帧结果直接作为严肃执法依据

## 4. 适用场景与边界

### 4.1 适用场景

- 固定机位监控视频
- 人体在画面中占比中等及以上
- 施工人员主要以站立、行走、轻度转身为主
- 安全帽和反光衣具有较明确颜色特征

### 4.2 不适用场景

- 人体极小、极远距离目标
- 遮挡严重，仅露出局部身体
- 大角度俯拍或仰拍导致人体结构区域明显变形
- 无反光衣或衣物颜色高度接近背景
- 极端强反光、过曝、雨雾严重影响颜色判断

## 5. 总体方案

整体链路如下：

1. 使用通用 YOLO 模型检测画面中的 `person`
2. 对每个 `person bbox` 生成人体结构化子区域
3. 基于子区域裁出：
   - `helmet_color_roi`
   - `vest_color_roi`
4. 在两个 ROI 内分别进行颜色分析
5. 输出每个人的识别结果
6. 将结果绘制到视频帧上，并保存视频与结构化记录

这套方案不要求 YOLO 直接检测“安全帽”或“反光衣”，而是把 YOLO 的作用限定为“稳定找人”。

## 6. 检测与识别逻辑

### 6.1 人体检测

#### 输入

- 单帧视频图像

#### 方法

- 使用通用 YOLO 检测模型识别 `person`
- 过滤低置信度的人体框
- 对明显异常的人体框做尺寸过滤

#### 输出

- `person bbox`
- `person_confidence`

### 6.2 安全帽颜色识别区域

基于人框按相对位置估计帽子颜色识别区域，而不是检测真实安全帽框。

建议初始规则：

- `helmet_x1 = px1 + 0.28 * person_w`
- `helmet_x2 = px2 - 0.28 * person_w`
- `helmet_y1 = py1 - 0.04 * person_h`
- `helmet_y2 = py1 + 0.16 * person_h`

设计原则：

- 横向只取人体上方居中区域，避免把肩膀和背景吃进去
- 纵向允许区域略高于人框顶部，以覆盖帽子抬高情况
- 不扩大到整个人体上半身，避免脸部和背景灯光过多进入

### 6.3 反光衣颜色识别区域

基于人框按相对位置估计反光衣颜色识别区域，重点覆盖躯干中上部。

建议初始规则：

- `vest_x1 = px1 + 0.20 * person_w`
- `vest_x2 = px2 - 0.20 * person_w`
- `vest_y1 = py1 + 0.22 * person_h`
- `vest_y2 = py1 + 0.72 * person_h`

设计原则：

- 只取躯干区域，不取腿部
- 尽量避开头部和手臂边缘
- 兼容反光衣前胸、后背、腰部大面积荧光区域

### 6.4 ROI 二次收缩

由于由人框估出来的区域仍然可能偏大，建议在实际颜色分析前再做一次内部裁剪：

- 帽子 ROI 只分析中间 `70% ~ 80%`
- 反光衣 ROI 只分析中间 `70% ~ 80%`

目标是进一步减少边缘背景、皮肤、头发、手臂和环境高亮对颜色判断的干扰。

## 7. 颜色识别方案

### 7.1 帽子颜色识别

建议优先识别以下帽子颜色：

- 白色
- 红色
- 黄色

方法建议：

- 将 ROI 转换到 `HSV`
- 计算各颜色掩码
- 统计颜色像素占比
- 使用主色占比或最大连通域占比做最终判定

建议规则：

- 红色达到阈值，判为红帽
- 黄色达到阈值，判为黄帽
- 白色达到阈值，判为白帽
- 都不满足时，判为 `unknown`

### 7.2 反光衣颜色识别

建议优先识别以下颜色：

- 黄绿色反光衣
- 橙色反光衣

方法建议：

- 同样基于 `HSV`
- 重点识别高饱和黄绿、橙色主区域
- 对低饱和高亮区域单独过滤，避免把白色反光条当成衣服主色

建议输出：

- `vest_color = yellow_green / orange / unknown`

### 7.3 亮度与反光处理

夜间工地场景建议增加轻量预处理：

- 只对 ROI 做 `CLAHE`
- 或轻微 gamma 提亮
- 对极高亮区域做掩码过滤

这样可以减少灯光过曝和反光条对颜色统计的影响。

## 8. 结果输出

建议每个人输出一条结构化结果：

```text
frame_index
frame_person_index
person_box
person_confidence
helmet_roi
helmet_color
helmet_color_score
vest_roi
vest_color
vest_color_score
role_label
decision_reason
```

字段说明：

- `frame_person_index`：当前帧内的人体序号，不表示跨帧稳定 ID
- `helmet_roi`：帽子颜色识别区域
- `vest_roi`：反光衣颜色识别区域
- `helmet_color`：帽子颜色分类结果
- `vest_color`：反光衣颜色分类结果
- `role_label`：业务标签，可按规则映射
- `decision_reason`：记录关键判定依据，方便调试

## 9. 业务映射建议

如果后续要映射管理人员、工作人员，可采用规则映射：

- 白帽 + 黄绿色反光衣 -> 管理人员
- 红帽 + 黄绿色/橙色反光衣 -> 工作人员
- 仅识别到帽子颜色但衣服颜色不稳定 -> 暂不下最终身份结论
- 任一关键颜色无法稳定识别 -> `unknown`

原则上：

- 颜色识别结果是视觉层结论
- 身份映射是业务层结论

两层不应混为一个结果，避免后续规则调整困难。

## 10. 视频输出要求

输出视频应支持直接可视化查看，建议包括：

- 人框
- 帽子 ROI
- 反光衣 ROI
- 帽子颜色标签
- 反光衣颜色标签
- 可选的调试文字，如颜色占比

输出目标：

- 便于人工快速验证是否裁区合理
- 便于判断误检是出在“检人”还是“颜色识别”

## 11. 参数设计建议

建议将以下参数暴露为可调配置：

- `person_conf`
- `helmet_roi_x_ratio`
- `helmet_roi_y_ratio`
- `vest_roi_x_ratio`
- `vest_roi_y_ratio`
- `red_threshold`
- `white_threshold`
- `yellow_threshold`
- `yellow_green_threshold`
- `orange_threshold`
- `clahe_enable`
- `gamma_value`

这样可以针对不同机位单独调优，而不必改代码逻辑。

## 12. 风险与问题

### 12.1 方案优点

- 不需要专门训练安全帽检测模型
- 不需要专门训练反光衣检测模型
- 只依赖“检人”这一条成熟能力
- 工程实现简单，调试直观
- 适合作为第一阶段快速验证方案

### 12.2 主要风险

- 由人框估计出的帽子区域不一定总是准确
- 人体侧身、弯腰、骑车时区域偏移会变大
- 反光衣区域容易混入手臂、背景或车辆
- 夜间高亮和反光条会影响颜色判断
- 多人重叠时，人框本身可能不稳定

### 12.3 风险应对

- 先做调试版输出，观察 ROI 是否合理
- 加入 ROI 可视化
- 加入短时投票，避免单帧误判
- 对不同机位单独调参数

## 13. 验收指标

建议至少从以下指标评估方案是否可用：

- 人体检测召回率
- 帽子颜色识别准确率
- 反光衣颜色识别准确率
- `unknown` 占比
- 单帧处理耗时
- 视频整体 FPS

建议在当前机位标注一批样本进行评测，覆盖：

- 正常站立
- 行走
- 多人靠近
- 夜间反光
- 遮挡

## 14. 实施建议

建议按三步推进：

### 第一步：先做检人 + ROI 可视化

- 只输出人框、帽子 ROI、反光衣 ROI
- 暂时不做复杂业务映射
- 先验证裁区是否合理

### 第二步：加颜色识别

- 帽子颜色分类
- 反光衣颜色分类
- 输出调试数值

### 第三步：加时序稳定

- 同一目标做 3 到 5 帧投票
- 降低夜间抖动和偶发误判

## 15. 结论

本方案适合作为当前阶段的第一版 PRD 方案。它以“通用 YOLO 检人”为基础，不追求一次性解决所有安全帽和反光衣检测问题，而是通过“人体结构化裁区 + 颜色规则识别”快速建立可运行、可调试、可验证的闭环。

对于当前固定机位夜间工地场景，这是成本最低、落地最快、最适合作为基线版本的方案。后续如果颜色 ROI 稳定性仍不足，再考虑升级为：

- `person + helmet + vest` 检测
- 或直接训练带颜色类别的专用 PPE 模型
