# PRD：优化方案 v1（姿态躯干反光衣识别）

## 1. 文档说明

本文档定义当前阶段的正式方案为“优化方案 v1”。原 `v3` 思路，即“仅基于通用 YOLO 人框按固定比例同时估计安全帽区域和反光衣区域”的方案，现阶段放弃，不作为主实现方向。

本次调整后的核心思路是：

- 继续使用通用 YOLO 模型识别画面中的人
- 安全帽颜色识别仍以人框顶部估计区域为主
- 反光衣颜色识别不再仅依赖固定比例人框裁剪
- 改为引入姿态识别结果，使用人体关键点估计“躯干区域”作为反光衣颜色识别区域

目标是提升反光衣颜色识别的稳定性，减少因人框过大、姿态变化、手臂遮挡、骑车姿态和背景混入带来的误判。

## 2. 背景与问题

当前场景为固定机位夜间工地视频，人物存在以下特点：

- 多人靠近站立
- 行走、侧身、转身、骑电动车等姿态变化明显
- 夜间强反光、过曝和地面反射较强
- 反光衣主要分布在上半身躯干区域，但整个人框中还包含头部、手臂、裤子、车辆和背景

若仅使用人框固定比例去估计反光衣颜色区域，存在以下问题：

- 人框裁区过大，容易混入手臂和背景
- 人体侧身时，反光衣真实区域会偏离固定裁区
- 骑车或坐姿时，固定比例区域会吃进车辆
- 多人靠近时，固定裁区容易把邻近人员的衣物颜色带入

因此，反光衣颜色识别需要更贴近人体结构，而不是只依赖人框比例。

## 3. 产品目标

### 3.1 核心目标

- 使用通用 YOLO 模型稳定识别出人
- 使用姿态识别结果估计躯干区域
- 以躯干区域作为反光衣颜色识别的主要 ROI
- 对安全帽和反光衣分别输出颜色结果
- 为后续“管理人员/工作人员”业务映射提供更稳定输入

### 3.2 本阶段定义

本阶段方案定义如下：

- `person detection`：采用通用 YOLO
- `helmet color ROI`：继续使用人框顶部估计
- `vest color ROI`：采用姿态关键点估计的躯干区域

### 3.3 非目标

当前阶段不追求：

- 独立安全帽检测框
- 独立反光衣检测框
- 跨摄像头身份追踪
- 单帧绝对高精度执法判断

## 4. 总体方案

每一帧的处理流程如下：

1. 使用通用 YOLO 检测 `person`
2. 对每个 `person` 运行姿态识别，获取上半身关键点
3. 基于人框生成安全帽颜色识别区域
4. 基于姿态关键点生成反光衣颜色识别区域
5. 分别在两个 ROI 内做颜色分析
6. 输出每个人的颜色识别结果
7. 将结果写回当前帧并保存视频

## 5. 识别链路设计

### 5.1 人体检测

#### 输入

- 单帧视频图像

#### 方法

- 使用通用 YOLO 模型识别 `person`
- 对低置信度目标做过滤
- 对尺寸明显异常的人框做过滤

#### 输出

- `person_box`
- `person_confidence`

### 5.2 姿态识别

对每个已识别的人，使用姿态模型获取关键点。

优先关注以下关键点：

- 左肩
- 右肩
- 左髋
- 右髋

可选辅助点：

- 鼻子
- 颈部近似点

其中，肩部和髋部是反光衣躯干区域的核心锚点。

### 5.3 安全帽颜色识别区域

安全帽颜色识别区域继续采用“人框顶部区域估计”方式。

建议初始规则：

- `helmet_x1 = px1 + 0.28 * person_w`
- `helmet_x2 = px2 - 0.28 * person_w`
- `helmet_y1 = py1 - 0.04 * person_h`
- `helmet_y2 = py1 + 0.16 * person_h`

设计目标：

- 尽量只覆盖头顶和帽子区域
- 减少肩膀和背景进入帽子 ROI

### 5.4 反光衣颜色识别区域

这是本次优化的核心。

反光衣 ROI 不再使用固定比例的人框中上部裁区，而是优先使用姿态关键点估计人体躯干区域。

#### 5.4.1 主方案：使用肩部与髋部关键点估计躯干 ROI

当左肩、右肩、左髋、右髋关键点足够稳定时：

- 以左右肩连线确定躯干上边界
- 以左右髋连线确定躯干下边界
- 以肩部和髋部左右边界外扩一定比例，形成躯干矩形 ROI

建议初始逻辑：

- `torso_x1 = min(left_shoulder_x, left_hip_x) - 0.08 * person_w`
- `torso_x2 = max(right_shoulder_x, right_hip_x) + 0.08 * person_w`
- `torso_y1 = min(left_shoulder_y, right_shoulder_y) - 0.04 * person_h`
- `torso_y2 = max(left_hip_y, right_hip_y) + 0.06 * person_h`

该区域基本对应反光衣主要覆盖范围。

#### 5.4.2 兜底方案：关键点不稳定时退化为人框比例区域

当肩部或髋部关键点缺失时，允许回退到保守的人框比例区域：

- `vest_x1 = px1 + 0.22 * person_w`
- `vest_x2 = px2 - 0.22 * person_w`
- `vest_y1 = py1 + 0.22 * person_h`
- `vest_y2 = py1 + 0.68 * person_h`

但该方案仅作为兜底，不作为主识别方式。

#### 5.4.3 为什么用躯干 ROI

相比固定比例裁区，躯干 ROI 的优势是：

- 更贴近人体结构
- 更能适应侧身和轻度转向
- 更少吃进头部、裤子、车辆和背景
- 更适合识别反光衣主色

#### 5.4.4 反光衣区域过小时的优化逻辑

实际视频中，反光衣 ROI 往往会因为以下原因显得偏小：

- 人物距离较远，整个人体框本身就较小
- 人体侧身、转身或骑车时，可见反光衣面积变窄
- 姿态关键点偏保守时，躯干 ROI 容易只落在衣服中间一条区域

为避免 ROI 过小导致颜色统计不稳定，补充如下优化逻辑：

- 主方案生成 `torso ROI` 后，允许在左右方向做小幅外扩，优先保证覆盖反光衣主体宽度，而不是只截取中间窄条
- 上边界允许略微贴近肩线下方，确保把胸前反光衣主体纳入，但仍需避免吃进头盔和脸部
- 下边界允许适度延伸到髋部下方少量区域，优先保证整块背心主体完整，但不得明显进入大腿和车辆区域
- 当姿态四点可用但生成的 `torso ROI` 宽度或面积低于最小阈值时，可按人体框宽度做一次保守补宽
- 当人物过远、姿态点抖动或单帧 ROI 过小时，应结合相邻帧做时序平滑，避免单帧 ROI 忽大忽小
- 当姿态 ROI 连续多帧过小且不稳定时，允许回退到保守的人框比例区域，优先保证可识别，再追求精确

建议增加以下最小约束：

- `torso_roi_min_width`
- `torso_roi_min_height`
- `torso_roi_min_area`

当 `torso ROI` 低于上述阈值时，不应直接按当前 ROI 做最终颜色结论，而应优先执行“补宽 / 回退 / 时序平滑”逻辑。

## 6. ROI 质量控制

为了避免姿态点抖动导致 ROI 异常，建议加入以下控制规则：

- 关键点置信度低于阈值时，不使用该关键点
- 若肩宽或躯干高度明显异常，则视为无效躯干 ROI
- 躯干 ROI 面积低于最小阈值时，不做颜色识别
- 躯干 ROI 超出人框合理范围过多时，回退到兜底方案

建议对最终 ROI 再做一次内部收缩：

- 只分析躯干 ROI 中间 `75% ~ 80%`
- 进一步减少边缘干扰

## 7. 颜色识别方案

### 7.1 安全帽颜色识别

建议优先识别：

- 白帽
- 红帽
- 黄帽

方法：

- 在 `helmet_color_roi` 内做 `HSV` 颜色分析
- 统计颜色像素占比
- 使用主色占比或主连通域占比做判定

### 7.2 反光衣颜色识别

建议优先识别：

- 黄绿色反光衣
- 橙色反光衣

方法：

- 在 `vest_color_roi` 内做 `HSV` 颜色分析
- 过滤高亮低饱和区域，避免反光条干扰
- 识别高饱和黄绿与橙色主区域

补充判定原则：

- 白色不作为反光衣主色的判定依据
- 原因是各类反光衣通常都带有高亮反光条，反光条在夜间和补光场景下会稳定呈现白色或近白色
- 因此，若将白色直接计入反光衣主色判断，容易把黄绿色、橙色等反光衣误判成白色反光衣
- 反光衣颜色区分应只基于白色以外的主色区域，当前阶段重点使用黄绿色、橙色、红色等非白色主色做区分
- 白色区域仅可作为“反光条存在”或“高亮干扰区域”的辅助信息，用于掩码过滤、调试展示和质量分析，不参与最终 `vest_color` 主类别决策

建议具体实现为：

- 先通过低饱和高亮规则识别白色/反光条区域
- 将该区域从反光衣主色统计分母或候选主色区域中剔除
- 再在剩余有效区域内统计黄绿色、橙色、红色等非白色颜色占比
- 最终 `vest_color` 仅在非白色颜色集合中做判定；若非白色有效面积不足，则输出 `unknown` 或 `other`

### 7.3 夜间增强建议

只对 ROI 做轻量增强：

- `CLAHE`
- 轻微 gamma 提亮
- 高亮反光掩码过滤

这样可减少夜间灯光和反光带来的颜色漂移。

## 8. 输出结果

建议每个人输出一条结构化结果：

```text
frame_index
frame_person_index
person_box
person_confidence
pose_available
helmet_roi
helmet_color
helmet_color_score
vest_roi
vest_roi_source
vest_color
vest_color_score
role_label
decision_reason
```

字段说明：

- `pose_available`：当前人是否拿到了可用姿态关键点
- `vest_roi_source`：取值建议为 `torso_pose` 或 `fallback_person_ratio`
- `helmet_color`：帽子颜色结果
- `vest_color`：反光衣颜色结果
- `role_label`：业务层映射结果
- `decision_reason`：记录本次结论依据

## 9. 业务映射建议

后续如需映射为管理人员、工作人员，可采用业务规则：

- 白帽 + 指定反光衣颜色 -> 管理人员
- 红帽/黄帽 + 指定反光衣颜色 -> 工作人员
- 帽子颜色识别不稳定或反光衣颜色识别不稳定 -> `unknown`

原则上：

- 视觉识别层只给出颜色结果
- 业务规则层再给出身份标签

## 10. 方案优势

优化方案 v1 相比被放弃的 `v3` 纯人框比例方案，主要优势是：

- 反光衣 ROI 更贴近真实躯干区域
- 对行走、侧身、转身更有适应性
- 更少把腿部、手臂、车辆和背景纳入反光衣分析
- 对多人靠近和局部遮挡更稳
- 仍然保留整体工程复杂度可控

## 11. 风险与限制

### 11.1 风险

- 夜间姿态关键点本身可能抖动
- 骑车或多人遮挡时，髋部关键点可能不稳定
- 远距离小目标时，姿态点可能不足以稳定生成躯干 ROI
- 安全帽区域仍然是估计值，不是真实帽子检测框

### 11.2 风险控制

- 姿态 ROI 仅用于反光衣，不强行用于帽子
- 关键点不足时立即回退到比例区域
- 输出视频中必须可视化 `helmet_roi` 和 `vest_roi`
- 输出调试信息，便于人工检查 ROI 是否合理

## 12. 验收指标

建议至少评估以下指标：

- 人体检测召回率
- 姿态 ROI 可用率
- 帽子颜色识别准确率
- 反光衣颜色识别准确率
- `unknown` 占比
- 单帧耗时
- 视频整体 FPS

建议重点对比两组方案：

- 基线：纯人框比例反光衣 ROI
- 新版：姿态躯干反光衣 ROI

若新版在准确率和稳定性上明显更优，则正式采用优化方案 v1。

## 13. 实施建议

建议分三步推进：

### 第一步：先做 ROI 可视化

- 检人
- 跑姿态
- 输出帽子 ROI 和躯干 ROI
- 暂不做最终业务映射

先验证：

- 躯干 ROI 是否稳定落在反光衣上
- 骑车、侧身、遮挡时是否仍然可用

### 第二步：加颜色识别

- 帽子颜色识别
- 反光衣颜色识别
- 输出颜色占比和最终标签

### 第三步：加时序平滑

- 对同一个人连续多帧做投票
- 降低单帧抖动和偶发误判

## 14. 结论

当前正式推荐方案为：优化方案 v1。

其核心定义是：

- 使用通用 YOLO 识别人
- 安全帽颜色继续基于人框顶部区域识别
- 反光衣颜色识别改为基于姿态关键点估计的躯干 ROI

原 `v3`“纯人框比例同时处理帽子和反光衣”的方案，因反光衣区域稳定性不足，现阶段放弃，不作为主方案推进。

该方案在工程复杂度、可解释性和实际稳定性之间更平衡，更适合作为当前项目的正式 PRD 版本。
