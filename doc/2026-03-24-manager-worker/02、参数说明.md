# 参数说明

这份文档说明 `helmet_color_classify.py` 和 `run-helmet-classify.ps1` 当前支持的主要参数，以及这些参数分别影响哪一部分逻辑。为了便于理解，这里按“分类判定”“颜色阈值”“ROI 定位”“输入输出”“显示调试”几个维度来说明。

当前代码里，红色的定义已经放宽，不再只识别“很纯很亮的红”。默认阈值现在是：

- `H: 0 ~ 18` 或 `145 ~ 179`
- `S >= 75`
- `V >= 60`

这比之前更容易把偏暗、偏灰、偏粉一点的红帽区域识别成红色。

## 1. 当前最终分类规则

当前代码是二分类逻辑，不再输出“未知”。最终规则可以概括为：

- 红色比例达到阈值 -> `工作人员`
- 否则 -> `管理人员`

代码中的关键判断是：

```python
if red_ratio >= args.red_ratio_threshold:
    label = "worker"
else:
    label = "manager"
```

因此，真正直接影响“管理人员 / 工作人员”结果的核心参数主要是：

- `red_ratio_threshold`

## 2. 分类判定参数

### 2.1 `red_ratio_threshold`

- 作用：红色像素比例阈值
- 默认值：`0.15`
- 含义：帽子分析区域里，红色像素至少要占多少比例，才有资格被判为红帽

影响：

- 调高：更难判成 `工作人员`
- 调低：更容易判成 `工作人员`

适用场景：

- 如果你希望“只要红色占 15% 就算工作人员”，那这个参数应改到 `0.15`

### 2.2 `white_ratio_threshold`

- 作用：白色比例阈值
- 默认值：`0.18`
- 当前状态：在现在这版二分类逻辑里，没有参与最终分类

说明：

- 这个参数原本是给“白帽 / 红帽 / 未知”三分类准备的
- 现在最终逻辑只看“是否红帽”，所以这个参数目前只是保留，没有实际影响最终结果

## 3. 红白像素阈值参数

这些参数不直接决定最终标签，但会影响 `white_ratio` 和 `red_ratio` 的计算结果。

### 3.1 `red_h_low_max`

- 作用：红色低位 Hue 区间的上界
- 默认值：`18`

含义：

- HSV 色相里靠近 `0` 的这一段会被当作红色

影响：

- 调大：更多偏橙红、偏粉红的区域会被算成红色
- 调小：红色定义更严格

### 3.2 `red_h_high_min`

- 作用：红色高位 Hue 区间的下界
- 默认值：`145`

含义：

- HSV 色相里靠近 `179` 的这一段也会被当作红色

影响：

- 调小：更多偏洋红、偏紫红的区域会被算成红色
- 调大：红色定义更严格

### 3.3 `red_s_min`

- 作用：红色最小饱和度
- 默认值：`75`

含义：

- 只有足够“纯”的红色，才会被计入红色像素

影响：

- 调高：能减少粉红、灰红、脏红误检
- 调低：更容易把浅红、发灰的红也算进去

### 3.4 `red_v_min`

- 作用：红色最小亮度
- 默认值：`60`

含义：

- 太暗的红色不算红色像素

影响：

- 调高：减少暗红背景或阴影干扰
- 调低：夜间暗处的红帽更容易被识别到

### 3.3 `white_s_max`

- 作用：白色最大饱和度
- 默认值：`55`

含义：

- 白色必须接近“低饱和”

影响：

- 调小：只有更纯的白、灰白、高亮白才会被计入白色像素
- 调大：浅蓝、浅粉、偏灰亮色也可能被算成白色

### 3.4 `white_v_min`

- 作用：白色最小亮度
- 默认值：`170`

含义：

- 白色必须足够亮

影响：

- 调高：更严格，只保留高亮白
- 调低：普通浅灰、弱反光区域更容易被计入白色像素

## 4. ROI 定位参数

这些参数影响“头盔区域是否取得准”，进而影响后续颜色判断。

### 4.1 `keypoint_conf`

- 作用：关键点最小置信度
- 默认值：`0.35`

含义：

- 鼻子、眼睛、耳朵、肩膀等关键点只有超过这个阈值，才会被用来估算头盔 ROI

影响：

- 调高：ROI 更保守，减少飘框，但可能取不到头盔区域
- 调低：ROI 更容易生成，但也更容易偏移到脸、背心或背景

### 4.2 `person_conf`

- 作用：人物检测置信度阈值
- 默认值：`0.35`

含义：

- 低于这个置信度的人体框会被过滤掉

影响：

- 调高：误检的人更少，但可能漏掉远处、小人、夜间目标
- 调低：检测更全，但也更可能把不稳定的人体框送去做后续分类

### 4.3 `imgsz`

- 作用：推理输入尺寸
- 默认值：`960`

影响：

- 调大：人物框和关键点通常更稳定，头盔 ROI 可能更准
- 调小：速度更快，但头部关键点精度下降

## 5. 模型与设备参数

### 5.1 `model`

- 作用：指定 YOLO 姿态模型
- 默认值：`yolo11n-pose.pt`

影响：

- 模型越大，关键点一般更稳
- 但速度也更慢

建议：

- 夜景、小目标、头盔较小场景，优先尝试 `yolo11m-pose.pt`

### 5.2 `device`

- 作用：指定运行设备
- 默认值：空，表示使用默认设备

可选：

- CPU
- CUDA GPU

影响：

- 只影响速度，不直接影响分类逻辑

## 6. 输入输出参数

### 6.1 `source`

- 作用：输入视频路径

### 6.2 `output`

- 作用：输出视频路径
- 默认行为：自动保存到 `outputs/helmet-classify/<视频名>.mp4`

### 6.3 `codec`

- 作用：输出视频编码器
- 默认值：`mp4v`

说明：

- 如果当前编码器不可用，会自动回退到 `XVID` 和 `.avi`

## 7. 中文显示参数

### 7.1 `font_path`

- 作用：指定中文字体路径
- 默认值：空

说明：

- 如果不指定，会尝试在 Windows 常见字体中自动寻找：
  - `msyh.ttc`
  - `msyhbd.ttc`
  - `simhei.ttf`
  - `simsun.ttc`

### 7.2 `font_size`

- 作用：视频中文标签字号
- 默认值：`22`

影响：

- 只影响显示大小，不影响分类结果

## 8. 调试与可视化参数

### 8.1 `draw_helmet_box`

- 作用：在输出视频中画出估计的头盔 ROI

适用：

- 用于确认帽子区域有没有取准

### 8.2 `debug_text`

- 作用：把 `white_ratio` 和 `red_ratio` 一起写到视频标签里

适用：

- 用于快速看某一帧为什么被判成 `管理人员` 或 `工作人员`

### 8.3 `save_debug_artifacts`

- 作用：保存调试产物

输出内容通常包括：

- 带标注的采样整帧
- 每个采样检测的四宫格图
- `records.csv`

### 8.4 `debug_dir`

- 作用：调试输出根目录

默认：

- `outputs/helmet-debug/<视频名>/<运行时间戳>`

说明：

- 当前代码会按“每次运行一个独立子目录”保存调试结果
- 即使你多次处理同一个视频，`frames`、`panels`、`records.csv` 也不会混在一起
- 如果手动指定 `debug_dir`，它会被视为根目录，程序仍然会在其下自动创建时间戳子目录

### 8.5 `debug_sample_every`

- 作用：每隔多少帧采一个调试样本
- 默认值：`60`

### 8.6 `debug_max_samples`

- 作用：最多保存多少个调试样本
- 默认值：`80`

### 8.7 `max_frames`

- 作用：只处理前多少帧
- 默认值：`0`，表示处理完整视频

适用：

- 调参数时先跑前 30 帧或 100 帧，节省时间

## 9. PowerShell 启动脚本里直接能传的参数

`run-helmet-classify.ps1` 当前已经直接暴露这些参数：

- `Source`
- `Model`
- `Output`
- `PersonConf`
- `WhiteRatioThreshold`
- `RedRatioThreshold`
- `RedHLowMax`
- `RedHHighMin`
- `RedSMin`
- `RedVMin`
- `Imgsz`
- `FontPath`
- `FontSize`
- `DrawHelmetBox`
- `DebugText`
- `SaveDebugArtifacts`
- `DebugDir`
- `DebugSampleEvery`
- `DebugMaxSamples`
- `MaxFrames`
- `Device`

没有单独暴露但 Python 脚本支持的参数，比如：

- `--white-s-max`
- `--white-v-min`

可以通过脚本的额外参数透传。

## 10. 当前场景下最值得关注的参数

如果你的问题是：

- 红帽上有白字
- 背景有高亮干扰
- 只要红色占一定比例就想判工作人员

那优先级最高的是：

1. `red_ratio_threshold`
2. `red_s_min`
3. `red_v_min`
4. `keypoint_conf`

其中：

- `red_ratio_threshold` 决定红色至少要占多少
- `red_s_min` / `red_v_min` 决定“什么样的红”才算有效红色
- `keypoint_conf` 决定帽子框是否更稳
