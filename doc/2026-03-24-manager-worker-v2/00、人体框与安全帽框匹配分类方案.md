# 人体框与安全帽框匹配分类方案

本文档给出一个替代当前“根据人体关键点估计帽子 ROI 再做颜色分析”的方案。考虑到当前视频清晰度有限，鼻子、双眼、双耳、双肩等细粒度关键点通常不稳定，因此不再把人体关键点作为主锚点。新方案改为优先检测以下目标：

- 人体框 `person bbox`
- 头部框 `head bbox`
- 安全帽框 `helmet bbox`

然后通过空间匹配关系，先把头部归属到具体的人，再把安全帽归属到具体的头部和人，最后在安全帽框内部做颜色分类。这个方案的核心目标是解决当前帽子 ROI 偏大、偏低、容易把背景和脸部一起框进去的问题。

## 1. 方案目标

当前实现的主要问题是：

- 帽子区域是估计出来的，不是真实检测框
- ROI 容易包含背景高亮区域、脸部、脖子和头发
- 一旦 ROI 偏移，后续红白像素占比就会明显失真

新方案要达到的目标是：

- 用独立的安全帽检测框替代估计 ROI
- 只在帽子框内做颜色统计，减少背景干扰
- 明确判断“这个人是否戴了帽子”
- 在多人场景下尽量避免把别人的帽子匹配到当前人身上

## 2. 总体流程

每一帧的处理流程建议如下：

1. 检测所有人，得到 `person bbox`
2. 检测所有头部，得到 `head bbox`
3. 检测所有安全帽，得到 `helmet bbox`
4. 先做 `head -> person` 归属
5. 再做 `helmet -> head -> person` 归属
6. 匹配成功：
   - 判定该人“戴安全帽”
   - 在对应 `helmet bbox` 内做颜色分类
7. 匹配失败：
   - 判定该人“未戴安全帽”

输出结果可以分为两层：

- 第一层：是否戴帽
- 第二层：帽子颜色类别，例如红帽、白帽、黄帽

## 3. 为什么不用“人与帽子框有重合”直接判定

如果直接用“`person bbox` 和 `helmet bbox` 有重合”作为规则，误匹配会比较多，原因是：

- 人框本身很大，帽子只要压到人框边缘一点就可能被算进去
- 多人靠近时，一个帽子可能同时与两个人人框重合
- 在拥挤场景下，旁边人的帽子很容易被错配

因此推荐的规则不是“人与帽子框有重合”，而是：

- 帽子框中心点落在该人的 `head bbox` 或头部匹配区内
- 同时帽子框有足够比例位于该头部区域上方或内部

## 4. 推荐的匹配规则

### 4.1 头部锚定区域生成

考虑到当前视频里基本识别不了鼻子、双眼、双耳、双肩等细节，推荐把“头部”作为新的主锚定点，而不是依赖人体关键点。头部锚定区域建议有两层：

- 主方案：直接检测 `head bbox`
- 兜底方案：当 `head bbox` 缺失时，由 `person bbox` 上部裁出一个固定头部候选区

推荐优先级如下：

1. 如果存在 `head bbox`，直接把它作为匹配锚点
2. 如果不存在 `head bbox`，再退化为 `person bbox` 上部区域

当没有 `head bbox` 时，兜底头部区域可以定义为：

- `head_x1 = px1 + 0.24 * person_w`
- `head_x2 = px2 - 0.24 * person_w`
- `head_y1 = py1 - 0.04 * person_h`
- `head_y2 = py1 + 0.18 * person_h`

这样可以保证即使头部检测临时漏掉，也不会退回到整个人框做匹配。

### 4.1.1 `head -> person` 归属规则

对于每个 `head bbox`，先找到它属于哪一个人。推荐规则：

- `head center` 落在 `person bbox` 内
- `head bbox` 的底边位于 `person bbox` 上半部分
- `head bbox` 面积占 `person bbox` 面积的比例在合理范围内，例如 `0.01 ~ 0.12`

如果同时满足多个 `person bbox`，优先分配给：

- 头部中心点离人框上边最近的那个
- 或 `head bbox` 被该人框包含比例最高的那个

这一步完成后，每个人都有一个更可靠的头部锚点。

### 4.2 帽子与人的匹配条件

对于每个 `helmet bbox`，计算其与每个 `head bbox` 或兜底头部区域的关系，推荐使用以下组合规则：

- 条件 1：`helmet center` 位于 `head bbox` 的水平投影范围内，或距离 `head center` 足够近
- 条件 2：`helmet bbox` 与 `head bbox` 有明显重叠，或位于 `head bbox` 正上方的小范围区域
- 条件 3：`helmet bbox` 的底边不能明显低于 `head bbox` 底边太多
- 条件 4：`helmet bbox` 的宽度与 `head bbox` 宽度比例在合理范围内，例如 `0.6 ~ 1.8`

之所以不强制要求“帽子框必须完全落在头框内”，是因为很多安全帽检测框会比头框略大，并且常常位于头框上方。更合理的做法是把头框作为锚点，允许帽子框与头框重叠或贴近。

一个简单可落地的判定可以写成：

- `abs(helmet_cx - head_cx) <= 0.6 * head_w`
- `helmet_y2 >= head_y1`
- `helmet_y1 <= head_y2`
- `helmet_y2 <= head_y2 + 0.35 * head_h`

满足以上条件后，再按距离和重叠比例打分。

### 4.3 多人多帽的一对一匹配

当一个帽子可能匹配多个人，或一个人附近有多个帽子时，需要做一对一分配。推荐方法：

- 先计算所有候选匹配对的分数
- 分数可由以下项组合：
  - 帽子中心点到头部中心点距离
  - 帽子框与头框的重叠比例
  - 帽子框是否位于头框正上方
  - 帽子框面积是否在合理范围内
- 按分数从高到低排序
- 使用贪心策略依次分配，已被占用的人和帽子不再重复匹配

如果后续场景更复杂，也可以换成匈牙利算法做全局最优分配。

## 5. 帽子颜色分类建议

新方案下，颜色分析不再使用估计头部 ROI，而是直接对 `helmet bbox` 做分析。这样可以显著减少背景混入。

颜色分类建议如下：

1. 从 `helmet bbox` 中再裁一个更保守的内部区域
2. 只分析中间 `70% ~ 80%` 区域，去掉边缘
3. 转到 HSV 或 Lab 空间
4. 计算红、白、黄等颜色的掩码
5. 使用最大连通域或主色占比做最终判定

建议不要直接用整个检测框面积作为分母，而应优先使用：

- 帽子主体掩码面积
- 或颜色连通域面积

这样可以进一步降低背景、反光和边缘噪声的影响。

## 6. 推荐的数据结构

建议每个人输出一条结构化结果，示例字段如下：

```text
person_id
person_box
head_box
has_helmet
helmet_box
helmet_match_score
helmet_color
helmet_color_score
```

其中：

- `has_helmet` 用于表示是否匹配到了帽子
- `head_box` 保存最终归属到该人的头部框
- `helmet_box` 保存最终归属到该人的安全帽框
- `helmet_color` 保存颜色分类结果

## 7. 这个方案的优势

相比“姿态点估计帽子 ROI”的方案，这个方案的优点更直接：

- 帽子框来自检测器，定位通常更准
- 颜色分析只在帽子框内部进行，背景污染更少
- 是否戴帽和帽子颜色两个问题被拆开，逻辑更清晰
- 更适合扩展到红帽、白帽、黄帽、蓝帽等多类别
- 更适合多人场景下做明确归属

## 8. 这个方案的风险和前提

该方案也有明显前提，需要提前说明：

- 必须有可用的头部检测模型，或者能从现有模型中稳定输出 `head bbox`
- 必须有可用的安全帽检测模型
- 远距离小目标时，帽子检测可能漏检
- 远距离或模糊场景下，头部检测也可能漏检
- 遮挡严重时，帽子框可能不稳定
- 如果头部检测或帽子检测质量差，后续匹配和颜色分类都会受影响

因此，该方案能否比当前方案更稳，关键取决于 `head detector` 和 `helmet detector` 的质量，尤其是：

- 头部小目标检测能力
- 小目标检测能力
- 夜间和模糊场景下的召回率
- 红白黄等不同颜色安全帽的检测一致性

## 9. 推荐实施顺序

建议分三步落地，而不是一次性全部重做：

### 第一步：先接入三类目标或两级检测

- 保留现有人检测
- 新增头部检测
- 新增安全帽检测
- 先只输出“人框 + 头框 + 帽子框 + 匹配关系”
- 暂时先不改颜色分类逻辑

先验证两个问题：

- 头部能不能稳定检出
- 帽子能不能稳定检出
- 头部和帽子能不能稳定匹配到正确的人

### 第二步：把颜色分析切换到帽子框

- 不再使用当前估计头部 ROI 做颜色判断
- 改为直接在 `helmet bbox` 内统计颜色
- 加入中心区域裁剪和连通域过滤

### 第三步：加时序稳定

对视频场景，建议加入短时跟踪或时序平滑：

- 同一人的帽子框做滑动平均
- 连续多帧匹配成功后再确认“戴帽”
- 颜色结果做 3 到 5 帧投票

这样可以减少单帧抖动和偶发误检。

## 10. 推荐结论

推荐把当前方案升级为：

- `person detector` 检测人
- `head detector` 检测头部
- `helmet detector` 检测安全帽
- 先做 `head -> person` 归属
- 再用头部框约束做 `helmet -> head -> person` 匹配
- 在 `helmet bbox` 内完成颜色分类

这个方案比“直接看人框和帽子框是否重合”更合理，也比“用人体关键点估计帽子 ROI”更稳。对于当前视频里关键点难以可靠识别的情况，用“头部框作为锚定点”会比“鼻子、双眼、双耳、双肩关键点”更可落地。对于当前“帽子框把很多不属于帽子的地方框出来”的问题，这个方向更值得投入。
